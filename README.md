# Agentic Engineering

This repository contains my plans, notes, skills, tools, and machine setup for adopting an agentic engineering workflow.

The goal is to gradually improve how I use AI agents alongside my existing software development workflow.

## Objectives

- Integrate AI agents into my sandbox development workflow
- Keep track of tools, configurations, and experiments
- Document useful prompts and workflows
- Build reusable skills and instructions for AI agents
- Review what works and what does not

## Current Setup

- macOS
- Neovim
- Git
- GitHub
- Terminal-based development workflow

More tools and configurations will be added as I explore different agentic engineering approaches.

## Repository Structure

```text
agentic-engineering/
├── README.md
```

### plans

Goals, learning plans, and future improvements.

### skills

Reusable instructions or skills that AI agents can follow.

### setup

Machine setup, tool installation, configuration files, and environment notes.

### prompts

Useful prompts for planning, coding, reviewing, debugging, and documentation.

### workflows

Documented development workflows that use AI agents.

Examples:

- Starting a new feature
- Investigating a bug
- Reviewing a pull request
- Writing tests
- Refactoring code
- Creating documentation

### notes

General notes, lessons learned, and useful references.

### experiments

Small experiments with different AI tools, agents, prompts, and workflows.

## Initial Plan

1. Learn the basic concepts of agentic engineering.
2. Choose an AI coding tool to test.
3. Integrate it with the existing terminal and Neovim workflow.
4. Start with small and low-risk development tasks.
5. Document successful prompts and workflows.
6. Create reusable skills and project instructions.
7. Review the workflow regularly and improve it.

## Guiding Principles

- Understand generated code before using it.
- Keep changes small and reviewable.
- Use version control for every meaningful change.
- Do not give agents unnecessary access to secrets or production systems.
- Verify commands before running them.
- Treat AI output as a suggestion, not automatically as the correct answer.
- Document useful discoveries for future reference.

## Status

This repository is currently in its early stage.

The structure, tools, and workflows will evolve as I gain more experience with agentic engineering.
